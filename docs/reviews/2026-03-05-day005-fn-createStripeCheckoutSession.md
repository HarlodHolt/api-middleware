## Target Summary

`createStripeCheckoutSession()` builds the Stripe Checkout Sessions API form payload, enforces request timeout via `AbortController`, and classifies Stripe/network failures into structured return variants for the order-creation path.

The function is one of the financially sensitive edges in the platform because it is the exact point where an order transitions from internal record state to an external payment workflow.

### A4. Data Flow Summary

The function receives precomputed arguments (`orderId`, `totalCents`, `customerEmail`, redirect URLs, timeout). It constructs a `application/x-www-form-urlencoded` Stripe request with one synthesized line item and an idempotency key derived from order ID. It returns a tagged result object:

- success: `{ ok: true, sessionId, url }`
- failure: `{ ok: false, status, errorCode, error }`

No database writes occur inside this function.

## C. Security Review

### C1. Authentication

Not handled directly here; inherited from route-level middleware. Current `/api/orders` path is HMAC-protected by default route policy.

### C2. Authorisation

Not handled at function scope; depends on caller route controls.

### C3. Input Validation

Function assumes validated inputs. It does not independently validate:

- `customerEmail` format
- redirect host integrity
- positive `totalCents` (it clamps to `>=0` but does not reject zero)

These validations are expected to be done by caller logic.

### C4. Injection Risks

No SQL execution. Form payload is generated by `URLSearchParams`; no header/body injection issue identified.

### C5. SSRF / Open Redirect

**Finding (REVIEW-005-001): allowlist trust can still be derived from request input.**

Although redirect URLs are checked before function call, the allowlist source is currently:

`configuredSiteBase = c.env.SITE_BASE_URL || site_base_url_from_request || ""`

If `SITE_BASE_URL` is unset, request-supplied `site_base_url` can define the hostname used by validation. That weakens the intended "server-owned allowlist" boundary and allows signed non-storefront callers to steer Stripe redirects to arbitrary HTTPS origins.

### C6. Webhook Abuse

Not applicable.

### C7. Rate Limiting

Not inside this function; upstream route middleware applies global worker rate limiting and storefront proxy rate limiting.

### C8. PII Handling

`customer_email` is sent to Stripe intentionally. Function does not log PII.

### C9. Secrets Handling

`secretKey` is only used in the Authorization header and is not returned in output.

### C10. Edge / Runtime Exposure

Uses Worker-safe APIs (`fetch`, `AbortController`, `URLSearchParams`).

### C11. Multi-Tenant Leakage

Not applicable (single-tenant deployment model).

### C12. Abuse Vectors

1. Redirect steering when server allowlist env is missing (see C5).
2. Poor upstream error shape extraction reduces operability during abuse/incident triage (`json.error` object is stringified to `[object Object]`).

## D. Observability and Operations

### D1. Logging Completeness

This function does not log directly; caller emits `orders.stripe.session_failed` and `orders.stripe.session_malformed_success`.

### D2. Error Handling

Typed failure modes exist (`stripe_timeout`, `stripe_network_error`, `stripe_upstream_error`, `stripe_malformed_success`).  
However, `stripe_upstream_error` currently extracts `String(json.error || json.message ...)`, which can collapse rich Stripe error objects into `[object Object]`.

### D3. Retry Logic

No internal retry (correct for payment initiation). Stripe idempotency key prevents duplicate checkout sessions for same order ID.

### D4. Performance Hotspots

External Stripe round-trip only; timeout guard is in place.

### D5. Failure Scenarios

| Scenario | Outcome |
|----------|---------|
| Stripe timeout | Returns `{ ok: false, status: 504, errorCode: "stripe_timeout" }` |
| Stripe DNS/network failure | Returns `{ ok: false, status: 502, errorCode: "stripe_network_error" }` |
| Stripe non-2xx with object error body | May return low-signal string (`[object Object]`) |
| Stripe 2xx without id/url | Returns `stripe_malformed_success`, caller keeps order in manual/pending |
